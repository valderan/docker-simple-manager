# Docker Simple Manager — FAQ и справка

Документ содержит основные сведения по использованию приложения, разбитые на тематические разделы. Используйте поиск в диалоге справки, чтобы быстро найти нужную информацию.

## 1. Основные сведения

- **Назначение**: управление локальными и удалёнными Docker-инстансами, контейнерами, образами, томами и историями сборок.
- **Поддерживаемые ОС**: Linux и macOS.
- **Рабочая директория**: `~/.dsmanager`, содержит конфигурации, проекты и логи.
- **Язык интерфейса**: русский или английский (переключается в настройках).
- **Темы**: светлая, тёмная и системная.

## 2. Главное окно

- **Панель подключения**: выбор активного соединения, кнопка обновления (по умолчанию `F5`).
- **Дашборд**: количество соединений и проектов.
- **Вкладки**:
  - `Контейнеры` — управление контейнерами.
  - `Образы` — список образов.
  - `Тома` — подключённые тома.
  - `Сборки` — история сборок (при наличии Docker Buildx).
- **Footer**: статус движка, системные метрики и терминал.
- **Горячие клавиши вкладок**: `1`–`4` переключают вкладки (настраивается в параметрах).

## 3. Менеджер соединений

- Создание локальных и удалённых соединений (SSH).
- Тестирование соединений, хранение статуса и комментариев.
- Возможность отмечать соединение по умолчанию и списка автоподключений.
- Вкладка соединений вызывается через меню или горячую клавишу.

## 4. Менеджер проектов

- Создание проектов, привязанных к соединению, описание, команда/путь, теги, приоритет.
- История запусков, экспорт/импорт конфигураций.
- Быстрый запуск последнего проекта через горячую клавишу.

## 5. Управление Docker объектами

- **Контейнеры**: запуск, пауза, перезапуск, удаление, просмотр логов и описания, запуск консоли.
- **Образы**: просмотр списка, удаление выбранного образа.
- **Тома**: просмотр подключенных томов, удаление, копирование точки монтирования.
- **Сборки**: отображение истории buildx (при наличии плагина).

## 6. Консоль контейнера

- Использует встроенную оболочку или системный терминал (как настроено).
- Команда для запуска задаётся в настройках (`/bin/sh` по умолчанию).

## 7. Логи приложения

- Доступны через меню параметров.
- Фильтры по файлу, уровню, диапазону дат и поисковой строке.
- Двойной клик по строке открывает отдельное окно с полным текстом и кнопкой копирования.
- Поддерживаются экспорт, удаление строк и очистка файла.

## 8. Справка

- Открывается из меню параметров, кнопки в настройках или по `F1`.
- Загрузка локального markdown-файла (этот документ) с поддержкой форматирования и поиска.

## 9. Настройки приложения

- **Основные**: язык, тема, сохранение состояния окна, интервалы обновления, таймауты соединений, системные метрики, консоль контейнера.
- **Логирование**: включение/отключение, уровень, размер файла, количество архивов.
- **Внешний вид**: цвета светлой/тёмной тем, акценты, шрифты (с возможностью сброса).
- **Горячие клавиши**:
  - Отображает список действий (меню, вкладки, обновление, справка и др.).
  - Для изменения комбинации используется специальный диалог: нажмите нужные клавиши, затем «Готово».
  - Поддерживается сброс к значениям по умолчанию.

## 10. Горячие клавиши по умолчанию

| Действие | Комбинация |
|----------|------------|
| Менеджер соединений | `Ctrl+Alt+C` |
| Проверить соединение | `Ctrl+Alt+T` |
| Менеджер проектов | `Ctrl+Alt+P` |
| Настройки | `Ctrl+Alt+S` |
| Логи | `Ctrl+Alt+L` |
| Справка | `F1` |
| Обновить данные | `F5` |
| Вкладки 1–4 | `1`, `2`, `3`, `4` |
| Выход | `Ctrl+Q` |
| Следующая вкладка | `Ctrl+Tab` |
| Предыдущая вкладка | `Ctrl+Shift+Tab` |
| Запуск последнего проекта | `Ctrl+Alt+R` |

Каждую комбинацию можно изменить на вкладке «Горячие клавиши».

## 11. Частые вопросы

1. **Не отображаются сборки** — установите Docker Buildx (см. инструкции во вкладке «Сборки») и перезапустите приложение.
2. **Не подключается удалённый Docker** — проверьте SSH-доступ, путь к ключу и таймаут в настройках.
3. **Не видно контейнеров** — убедитесь, что выбрано активное соединение и нажмите `F5`.
4. **Недоступен терминал** — включите «Использовать системную консоль» или задайте корректную оболочку.
5. **Не работают горячие клавиши** — перепривяжите их и проверьте, чтобы комбинации не конфликтовали между собой.

## 12. Полезные советы

- Используйте разные цвета тем для различимых интерфейсов.
- Храните проекты в отдельной папке `~/.dsmanager/projects` для быстрого экспорта.
- Регулярно проверяйте логи ( `Параметры → Логи` ) для выявления ошибок подключений.
